# CLAUDE.md — baseballab.com プロジェクト行動指針

LLM によるコーディングでありがちなミスを減らすための行動指針。

**トレードオフ:** これらの指針は、スピードよりも慎重さを優先する方向にバイアスをかけている。些細なタスクについては自分の判断を使うこと。

## 1. コーディングの前に考える

実装する前に:
- 自分の前提を明示的に述べる。不確実なら質問する。
- 複数の解釈が成り立つ場合は、それらを提示する — 黙って 1 つを選ばない。
- よりシンプルなアプローチがあるなら、そう言う。
- 何か不明瞭な点があれば、立ち止まる。質問する。

## 2. シンプルさを最優先に

- 求められた以上の機能は作らない。
- 単一用途のコードに抽象化は持ち込まない。
- 200 行書いて、それが 50 行で済むなら、書き直す。

## 3. 外科的な変更

- 隣接するコード、コメント、書式を「改善」しない。
- 壊れていないものをリファクタリングしない。
- 自分の変更が孤立した要素を生んだとき: 未使用になった import/変数/関数を削除する。

## 4. ゴール駆動の実行

- タスクを検証可能なゴールに変換する。
- 複数ステップのタスクでは、簡潔な計画を述べる。

## 成績数値の表示ルール（ハード固定）

| カテゴリ | 対象指標 | 表示形式 | 例 |
|---|---|---|---|
| 打率系（3位まで固定） | AVG / OBP / SLG / OPS | `0.000`（小数第3位固定・四捨五入） | `0.320`、`0.000` |
| 防御率系（2位まで固定） | ERA / WHIP | `0.00`（小数第2位固定・四捨五入） | `3.45`、`0.00` |

- **0でも固定桁数を表示**：値が 0 でも `0.000` / `0.00` を表示し、`0` や空欄にしない
- データが存在しない（null / undefined）場合のみ `—` を返す
- フォーマット関数 `f3()` / `f2()` / `fmtCard()` / `fmtVal()` / `fmtL()` を frontmatter に定義して使用

## NPB ステージング固有の指針

### このフォルダの役割
`_npb_deploy/stage/` は NPB 追加機能のステージングエリア。
`deploy_npb.bat` を実行すると webapp/ にコピーされて本番デプロイされる。

### 使用する lib ファイル
- `src/lib/npb.ts` — NPB データローダー（`loadPlayers()`, `loadStandings()`, `loadLeaders()`, `loadMeta()`）
- データは `data/npb/` 配下の JSON から読み込む（ビルド時）

### データ取得スクリプト
```bash
python scripts/fetch_npb.py --mode all  # 全データ取得（4〜5分）
python scripts/fetch_npb.py --mode test # 接続テスト
```

### デプロイコマンド
`_npb_deploy/deploy_npb.bat` をダブルクリック（または PowerShell から実行）

### 新規ページ追加時の注意
1. `src/pages/npb/` 配下に作成
2. `deploy_npb.bat` の `git add` リストに追加が必要
3. `../../layouts/Layout.astro` を import して使用
4. データ読み込みは `../../lib/npb` から
